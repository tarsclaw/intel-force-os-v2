# Migration: brain_graph

Adds per-tenant brain graph storage.

## What it creates

- `ops.brain_graphs` — one row per tenant, holds the full graphify output as JSONB
- `ops.brain_edge_reviews` — review history for INFERRED edges (Phase 5)
- `ops.BrainGraphStatus` enum: `PENDING / BUILDING / READY / FAILED / STALE`
- `ops.EdgeDecision` enum: `APPROVED / REJECTED / UNCERTAIN`

## How to apply

**Development:**
```bash
pnpm --filter @intelforce/db prisma db push
```

**Production:**
```bash
pnpm --filter @intelforce/db prisma migrate deploy
```

## Reversibility

The down migration is:
```sql
DROP TABLE "ops"."brain_edge_reviews";
DROP TABLE "ops"."brain_graphs";
DROP TYPE "ops"."EdgeDecision";
DROP TYPE "ops"."BrainGraphStatus";
```

Drop is safe: tenant data is not touched. Worst case is losing computed brain
graphs, which can be regenerated by re-running the brain builder against the
existing handbook + source files.
